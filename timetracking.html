タイムトラッキング

Posted on Sun, Sep 15, 2019 TrackingTime Toggl TickTick Notion

普段の生活の中で、自分が何にどのくらいの時間を把握していますか？

覚えていなくとも記録が残っていれば、必要になった時に見直すことができます。

最近では家計簿アプリが使いやすくなり、資産管理をする人も増えてきました。しかし自分の時間を記録で追えるようにする人はあまり多くないでしょう。

家計簿アプリの場合、店舗やオンラインショップで決済した後、手打ちまたはアプリと連携したカード会社からの通知により資産の推移を記録することができます。1日で複数回の決済があったとしても、それでも回数も少なく、さらに連携による自動化のためあまり苦労にはならないでしょう。

一方でタイムトラッキングを考えると、タスクを変更するたびに操作が発生します。スマートフォンやPCでタスク名と開始時刻・終了時刻の入力が必要です。測ろうと意気込んでもすぐにトラッキングを始められなければ、それだけでやる気を失ってしまいます。そのためできるだけアプリ内での画面遷移を減らすことが重要です。

私が使用しているタイムトラッキングアプリは「TrackingTime」と「Toggl Track」です。またToDoリストは「TickTick」を使用しています。

2つのタイムトラッキングアプリを使用するには理由があります。デスクワーク時とそれ以外の利用によって使用するアプリを分けています。

まず研究や仕事などデスクワークなどの場合PCを起動していることが前提となります。デスクワークではタスクの見落としなどを防ぐためToDoリストを使用しています。TickTickはTodoリストとしての機能やUIも素晴らしく非常に使いやすいタスク管理アプリです。特にタスクの説明欄であるメモではMarkdown記法が使用できるため重宝しています。TrackingTimeはブラウザ拡張の「Tracking Time | Button」を提供しています。このボタンはTickTickの各タスクのメモ欄にオーバーレイされます。またブラウザの拡張機能ボタンでは現在進行中のタスクを確認できます。TickTickのタスクのメモ欄のボタンを押すことでタスク名の設定を行わずタイムトラッキングが始められます。停止や一時停止したい場合もボタンを押すことで簡単にタイムトラッキングを止めることができます。

💡

TrackingTimeの無料プランは100 Tasks/monthまでとなっています。有料のProプラン($4.99/月)では作成できるTaskの数が無制限となります。私の方法では、100 Tasksを超えることが予測されます。まずはアカウントを作る際に14daysのProプランの無料オプションを選択し、使い勝手を確認してみてください。

デスクワーク以外の場面でもタイムトラッキングもTrackingTimeがベストアプリなのかというと、そうではありません。デスクワークの場合Todoアプリと組み合わせることでタスク名等の入力作業を省くことができました。しかしスマートフォンではブラウザ拡張機能を使えません。スマートフォンのTrackingTimeアプリは1操作でタイムトラッキングを開始することはできません。そのためToggl Trackを使用します。

Toggl Trackは非常にシンプルなアプリであり、スマートフォン・ブラウザ・PCアプリのどの環境でも操作性やUIは変わりません。Toggl Trackは開始ボタンを押すだけでタイムトラッキングを開始できます。またタスク名を入力する時間が無い場合も入力せずにタイムトラッキングを開始できます。

💡

TrackingTimeの有料プランを契約したくない方、ToDoアプリを普段使用していない人は、Toggl Trackのみでタイムトラッキングすることをおすすめします。

2つのアプリを場合によって使い分けるのは、できるだけ少ない操作で記録を行うためでした。ただ2つのアプリを使用するため、振り返るには2つのアプリのログを統合する必要があります。そのため統合するためのスクリプトを自分で作成しました。

matsunom/merge_timelog

このスクリプトは下記の2つのアプリのレポートファイルを組み合わせ、タイムエントリを昇順に並び替えます。 Toggle Track Detailed Report TrackingTime Report 扱える期間は次のようになっています。 そのためレポート(CSVファイル)を各アプリのウェブサイトからダウンロードする際は、上記の日付をレポート範囲として選ぶ必要があります。 またTrackingTimeでレポートを選択する際は、TASK・FROM・TO・DATEのカラムがレポートに含まれる必要があります。 それぞれのアプリのレポートのダウンロード方法は次のようになります。 Reports → Detailed → 期間の選択 TrackingTime Reports → TimeSheets → 期間の選択 ダウンロードしたレポートはinputディレクトリに移動させます。スクリプトを実行させるとCSVファイルがoutputディレクトリに出力されます。 pythonではなくpython3で実行してください。 このスクリプトは3つの引数を持ちます。 python3 merge_timelog.py 期間 開始時刻 エンコーディング 例) python merge_timelog.py today python merge_timelog.py today 8 utf8 期間は必ず指定が必要ですが、開始時刻とエンコーディングは必要に応じで指定してください。開始時刻とエンコーディングの順番は問いません。 期間 期間を省くことはできません。また以下の4つのみを受け付けます。 yesterdayは昨日1日のみを扱います。昨日の日付で出力ファイルを作成します。 yesterday~は昨日と今日の2日間を扱います。昨日の日付で出力ファイルを作成します。 todayは今日1日のみを扱います。今日のの日付で出力ファイルを作成します。 -tを選択した場合は、yyyy-mm-ddで日付を指定します。 例) python merge_timelog.py -t 2020-09-15 開始時刻 任意で開始時刻を設定できます。開始時刻を整数(0~23)で設定すると、開始時刻以降の時間帯のみを出力ファイルに書き込みます。デフォルトでは最初に6時間以上エントリー間に開きがあった場合、6時間の開きがあった後のエントリーから出力ファイルに書き込みます。 例) python merge_timelog.py 7 エンコーディング 任意でエンコーディングを指定できます。選べるのはutf8とp932(SHIFT-JIS)となっています。デフォルトではutf8が指定されています。Excelで開きたい場合は、p932を指定してください。 例) python merge_timelog.py cp932 日付指定、開始時刻、エンコーディングを同時に指定することができます。 例) python merge_timelog.py -t 2020-09-15 7 cp932 outputディレクトリに作成される'TimeLog_20200915.csv'CSVファイルには、2つのレポートファイルをマージし、開始時刻の昇順で並び替えられたものが書き込まれます。書き込まれるカラムはTask, From, Toの3項目となっています。時刻は24時間表記で書き込まれます。

https://github.com/matsunom/merge_timelog

私の場合、メモアプリの「Notion」で日記をつけており、そこに1日のタイムトラッキング結果を記載しています。そこで記載するのが、タスク名・開始時刻・終了時刻の3点です。スクリプトの使い方の詳しい説明は私のGitHubのプロジェクトページに記載されています。

はじめにで書いたように、私は日記にタイムトラッキングを組み込むことで、1日の進捗や成果が可視化され、前に進んでいることが実感できるようになりました。無力感に悩まされている方はタイムトラッキングをしてみてはいかがですか？
